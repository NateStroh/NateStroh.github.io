The Archon Project - UE4

Out now on itch.io!

The Archon Project is a third-person rougelite, action game focused on fighting other sentinels(robots), exploring environments and evolving your sentinel to be the apex predator. The Archon Project was built over the Spring 2021 semester by 9 students and involved many complex systems built in Unreal Engine 4. My main contributions were in engineering but I also played a role in game design and level design.

My Contributions:

  • Game Design and Game Balancing
  • Level Design: City Block, Desert Giant's Grave, Grass Stone Henge
  • Rapid protyping of combat abilities/traits
  • Map Generation: Built a custom hextile spawner and tileset editor of sorts that spawned custom hextiles (biomes). Uses level streaming to handle numerous tiles while keeping performant and keep state.
  • Encounter Spawning: A robust system built to spawn AI enemies for the game with the idea of spawning different combinations of or different AI as the game progresses to create difficulty rather than just increasing stats of enemies. Was also built with replayability in mind as it lets designers choose different combinations of AI to spawn with different chances, essentially randomizes enemy encounters the player will have.
  • HexLock System: System to keep player contained to a tile until they "Decode" the forcefield. Provides a immeadiate goal for the player and puts some control on the pacing of the game.

Familiar Souls - UE4 and Twitch

Familiar Souls is a exploration game in which the player must escape a maze and avoid the giant cat lurking in the maze built in ~3 weeks by ~10 students. However, there's a twist in that the game requires the player to be streaming and use Twitch chat to solve puzzles, survive and escape. In the video on the left you are seeing what Twitch chat can see, the player can only see the bottom screen and must rely on information present in the top screen. Additionally, Twitch chat can use chat commmands to guide the player through the maze and solve puzzles (like raise/lower platforms or doors).

My Contributions:

  • Game Design: Interesting challenge of a game designed specifically to interact with Twitch chat.
  • Twitch Integration: Twitch chat can do numerous commands to effect gameplay such as deciding what power-ups spawn, moving platforms, or opening doors. I was in charge of capturing this input from Twitch and add it to a vote system that would decide what would happen in game. I also created a system that would message Twitch chat with the commands they could use at that moment.
  • UI: I also did all of the UI programming for the game. The UI itself wasn't too complicated however Twitch chat could give a direction hint that was tied to the UI (left side of the player screen). The UI also incorporated a stream of Twitch chat so the player didn't have to look away from the screen to get information from the chat.

Darkroom - Unity

Darkroom is a more serious game in that this was a game designed to teach players how photography was once done, in a dark room! This vertical slice of a game takes players through the process of developing a photo and even provides some interesting information about Utah wildlife. Darkroom was also built with usability in mind as all you need is your phone to experience this AR game.

My Contributions:

  • Game Programming: Did a lot of interesting programming and design with this AR phone control in mind. For example, the player uses the gyroscope feature in their phone to explore the game.

Sharkbomber - Phaser

Sharkbomber is a 2D platformer that is about a Shark that must avoid mines and toxic goo in order to eat the fisherman on the surface and finally get some much needed peace! Sharkbomber is based on the classic arcade game Donkey Kong, but designed around the question of "What if Donkey Kong was underwater?". This interesting iteration on the classic really forced some interesting design decsisions as we modified the core mechanic and make it fun!

My Contributions:

  • Game Design
  • Game Programming
  • Game Systems

Bounce - Monogame

Bounce is a 2D platformer that is all about gaining momentum to clear jumps and defeat enemies. Certain enemies can only be destroyed by achieving max momentum and the player must balance speed and caution. Bounce was designed around an interesting premise in that we were assigned a random toy (a paddle ball in this case) and had to make a game based off of what makes that toy fun. In the case of the paddle ball, we decided that the pressure to keep building up momentum and keeping that ball bouncing was what made it fun and tried to capture that feeling in a game.

My Contributions:

  • Game Design
  • Game Programming
  • Game Systems: Had to write many of the systems we used ourselves. For example, I wrote the simple animation system our game used.
